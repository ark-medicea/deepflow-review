💬 Chat Widget — Floats as a compact card over the canvas (bottom-right of graph area, not the viewport). Overlays the graph without causing layout shift — nodes don't jump when chat opens/closes. Can be minimised to just the header bar. On screens < 1440px, opening chat auto-minimises the detail panel.
📋 Detail Panel (360px) — Slides in from the right when any node is clicked. Shows tabbed content: Overview, Sub-tasks, Comments, History. Displays dependency chains in both directions — critical for answering "what's blocking this?" Sub-tasks have inline checkboxes and running status indicators.
🔗 Edge Lines — Dashed lines = completed dependencies. Solid lines = active/in-progress flows. Muted lines = pending future connections. Colour follows the upstream node's status colour for instant visual tracing.
⬡ Hexagonal Avatars — AI agents use hexagonal avatars to visually distinguish them from human team members (circular). This is a key design decision — users can scan a graph and instantly see which tasks are human-driven vs AI-driven.

📏 Collapsed Sidebar (56px) — Shows only icons, maximising table width. The sidebar can be toggled between expanded (240px) and collapsed (56px) modes. In List View, collapsed is the default since users need column width, not project navigation.
🔽 Expandable Rows — Parent tasks show ▾ (expanded) or ▸ (collapsed) toggle. Child tasks are indented with extra left padding. This mirrors the graph's parent-child containment in a flat table format. Selected row gets a subtle blue background highlight.
🏷️ Priority Pills — Colour-coded: Critical (red), High (blue), Medium (grey). These use the same status colour system as the rest of DeepFlow. The "Blocked" and "Gate" inline badges provide at-a-glance status beyond the status dot.
📋 Detail Panel — In List View, the detail panel is hidden by default. Clicking a row opens it as a slide-over from the right (same 360px panel as Graph View). This keeps the table at full width until the user explicitly requests detail.

📊 Summary Stats — Three key numbers at a glance: tasks due today, blocked items, and items awaiting approval. These are the three questions Ravi asks every morning. Colour-coded to status severity — red for blocked, amber for due, purple for approval gates.
🏥 Health Meter (5 segments) — Visual at-a-glance health for each project. Green segments = on track, Amber = at risk, Empty = no data/poor. Calculated from: % completion vs expected timeline, overdue tasks, blocked tasks. Ravi can scan 10 projects in seconds without reading numbers.
✓ ▶ ✕ Status Counts — Done, In Progress, Failed — using the same colour language as everywhere else. Consistent with graph node status dots and list view status indicators. No learning curve between views.
🌈 New Workflow Button — Uses the brand gradient with box-shadow glow. Single primary CTA following the "Rule of One" — one clear action per screen. Positioned in the header so it's always visible without scrolling.
