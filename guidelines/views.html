💬 Chat Widget — Floats over the canvas as a compact card (bottom-right of graph area). No layout shift when opened/closed. Can be minimised to header only. On screens < 1440px, opening chat auto-minimises the detail panel.
📋 Detail Panel (360px) — Slides in from right when a node is clicked. Tabbed: Overview, Sub-tasks, Comments, History. Shows upstream and downstream dependencies — answers "what's blocking this?" and "what does this block?"

📊 Collapsed State (44px) — One-line summary: workflow name, progress bar with percentage, status count chips (coloured dots + numbers), 5-segment health meter, due date. Always visible between topbar and canvas. Click anywhere to expand.
📈 Expanded State (~220px) — Four-column dashboard: (1) Completion donut showing % breakdown by status, (2) Status breakdown bars with counts, (3) Upcoming due dates sorted chronologically, (4) Top assignees with task counts. Collapses back to one-line on click.
🔄 Level-Aware — The Hub Bar updates when you navigate levels. At L1 (ICAAP 2026), it shows aggregate stats for all workstreams. Drill into L2 (Capital Modelling) and it recalculates for just that workstream's tasks. Breadcrumbs and Hub Bar always stay in sync.
📱 Responsive — On tablet, the expanded view stacks into 2×2 grid. On mobile, it becomes a scrollable horizontal card strip. The collapsed bar always fits in one line.

🔍 Drill Trigger — Double-click an L1/L2 group zone or click the "Enter" icon on a parent node. The graph animates: zooms into the group, fades out sibling groups, replaces content with children. Breadcrumbs gain one level. Hub Bar recalculates.
🔙 Navigate Back — Click any breadcrumb level to zoom back out. Click "ICAAP 2026" in breadcrumbs to return to L1 view. Hub Bar recalculates to show aggregate stats for that level. The transition is a smooth 400ms zoom-out animation.
📊 Hub Bar Sync — Hub Bar always reflects the current navigation level. At L1: aggregate stats across all workstreams (62% complete, 18 tasks). At L2 Capital Modelling: only that workstream's tasks (45% complete, 8 tasks). The donut, status bars, due dates, and assignees all recalculate.

Responsive

Responsive Behaviour

How each view adapts across breakpoints. Desktop-first — mobile is for monitoring, not graph creation.

📊 Graph View

Desktop (≥ 1440px)All panels visible: expanded sidebar + hub bar + canvas + detail panel. Chat overlays graph. Both detail and chat can coexist.
Tablet (768–1023px)Sidebar hidden (hamburger). Hub bar collapses to one-line only. Detail panel becomes full-screen overlay. Chat is full-screen modal.
Mobile (< 768px)No graph canvas. Defaults to List View. Detail as bottom sheet. Chat as modal. Hub bar becomes a sticky header card.

📋 List View

Desktop (≥ 1440px)Full table with all columns. Sidebar collapsed. Detail slides from right on row click.
Tablet (768–1023px)Priority and Due Date columns hide. Table scrolls horizontally. Detail as overlay.
Mobile (< 768px)Card-based layout replaces table. Each task is a card. Tap for detail bottom sheet.

🏠 Workspace View

Desktop (≥ 1440px)2-column project card grid. Full stats bar. Sidebar expanded.
Tablet (768–1023px)2-column cards. Sidebar collapsed. Stats wrap to 2 rows.
Mobile (< 768px)1-column cards. No sidebar. Stats as swipeable row. Filters in dropdown.

🗂️ Gallery View

Desktop (≥ 1440px)3-column project grid. Template row visible. Full search bar with filters.
Tablet (768–1023px)2-column gallery grid. Templates scroll horizontally. Sidebar collapsed.
Mobile (< 768px)1-column gallery. Templates as swipeable cards. Search full-width.

📊 Hub Bar

Desktop (≥ 1440px)Full-width between topbar and canvas. Expanded: 4-column dashboard. Collapsed: one-line summary.
Tablet (768–1023px)Collapsed only (one-line). Tap to expand as 2×2 grid overlay.
Mobile (< 768px)Becomes a sticky card above the list. Shows progress + status counts. Tap to open full-screen metrics modal.
Key Decision: No Graph on Mobile. Dependency graphs with 20+ nodes are illegible at 375px. Mobile defaults to List View — same data, mobile-appropriate format. Users can browse, update status, and chat with AI. Graph editing is a desktop activity.
