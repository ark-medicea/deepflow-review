Sidebar Navigation

The sidebar appears in every view. In Option B (chat always on), it lives at 56px collapsed most of the time. Every icon must be instantly recognisable without a label. Design for collapsed first.

Information Hierarchy

Three zones, top to bottom: where you are → where you go → who you are.

🔝 Top — Identity & Context 🧭 Middle — Navigation 👤 Bottom — Utility & Profile
1
Org switcher at the very top. It sets everything — all data flows from this choice. Pick the wrong org, everything is wrong. First thing you see, always.
2
Dashboard → Tasks → Agents follows natural workflow. Overview → my work → my tools. Three nav items. No more.
3
Notifications, Settings, Profile at the bottom. Universal pattern (Slack, Linear, Notion, Discord). Users' muscle memory already knows to look there.
4
No "Workflows" or "Templates" as nav items. Templates live in Gallery. Workflows live inside Projects. Keep the sidebar lean — every extra icon is cognitive load at 56px.

Expanded View — 240px

Full labels, expandable project list, agent status. Auto-expands in Settings view only. Click-to-expand everywhere else (no hover-expand — it causes layout jank with the 320px chat panel).

Zone Breakdown

🔝 Org Switcher (top) — Logo/initials + org name + plan tier + dropdown chevron. Click opens dropdown showing all orgs with checkmark on current. "Create new org" at bottom. Switching orgs reloads all content with a brief fade transition.
🧭 Dashboard — Home view. Landing page with project cards, stats, attention items. Active state: blue background tint + 3px left accent bar.
🧭 Tasks & Projects — Expands inline to show pinned/recent projects (3–5 max). Each project shows a coloured status dot. "View all projects →" links to Gallery. The "+" button pins a new project.
🧭 Agents & Automations — Hexagon icon (matches AI avatar shape). Shows active agents with status dots: green pulse = running, grey = idle, amber = needs attention. "Manage all →" links to full agent management view.
👤 Notifications — Red count badge when unread. Click opens 320px notification panel (slides in from right, overlays detail panel). Grouped: "Task assigned", "Agent completed", "Approval needed".
👤 Settings — Navigates to settings view. Sidebar auto-expands to 240px in Settings since there's no chat context needed and it helps with the settings-nav pattern.
👤 User profile — Avatar + name + email. Click opens popover: profile, status, sign out.

Collapsed Behaviour

Tooltips — Every icon gets a tooltip on hover (200ms delay, right-aligned). Shows the label: "Dashboard", "Tasks & Projects", etc.
Flyout mini-panels — Hover on Tasks or Agents shows a flyout panel (right of sidebar) with the expandable content: project list or agent list. Like Slack's sidebar hover behaviour. Flyout disappears on mouse-out.
Notification dot (not number) — At 56px, a number badge is unreadable. The dot says "something's there". Full count visible in the notification panel or on expanded sidebar.
Agent running indicator — Small green pulsing dot on the Agents icon when any agent is actively running. Ambient awareness without clicking.
Active state — 3px left accent bar (brand blue) + subtle icon colour tint. Clearly visible even at icon-only size.
Org logo — If uploaded: show logo at 24×24, rounded. If not: first 2 letters on a gradient background.

Tab Bar Design

💬 Chat promoted to first-class tab — No persistent side panel on mobile. Chat needs to be one tap away at all times. This is the DeepFlow differentiator — AI assistance is always accessible.
⬡ Agents lives inside Tasks tab — As a secondary section or filter toggle. Doesn't need its own tab. Power users go there; casual users live in Tasks.
👤 "You" tab contains everything personal — Profile, org switcher, settings, integrations. Org switching flow: You tab → Org section at top → tap current org → bottom sheet with org list → select → reload.
🔔 Notification badge — Same red dot as collapsed sidebar. Tap opens full notification list.
No graph canvas on mobile. Defaults to List View. Tasks are card-based. Detail opens as bottom sheet. Graph editing is a desktop activity.

Org Switching

Org switching is a mode change, not navigation. When you switch orgs, everything reloads — projects, tasks, agents, chat history. Make the gravity of this action clear.

Dropdown contents: All orgs the user belongs to, with a checkmark on the active one. Each row shows: org logo/initials, org name, role (Admin/Member). "Create new organisation" at the bottom with a "+" icon.
Switch animation: Brief 300ms fade on the entire content area. Content reloads. Breadcrumbs reset to "Dashboard". Chat clears and shows org-scoped greeting.
Data isolation: Complete. No cross-org data leakage. Projects, tasks, agents, notifications, chat history — all org-scoped. The only cross-org data is the user's profile and global preferences.

Notification Panel

Click the bell → sliding panel from right (replaces or overlays detail panel). Not a separate page — notifications are contextual.

Grouped by type: "Task assigned to you", "Agent completed", "Approval needed", "Comment on your task", "Workflow status change". Each group is collapsible.
Each notification shows: Icon (type), title, description, timestamp (relative — "2h ago"), actor avatar. Click → navigates to the relevant task/project. Swipe to dismiss (mobile).
Mark all as read button at top. Individual dismiss on hover (desktop) or swipe (mobile).
Multi-channel settings: Per notification type, users configure which channels receive it: in-app bell, email, Slack, push (mobile). Settings UI shows a matrix — accessible from the notification panel header ("⚙ Notification settings") or from Settings page.

Key Design Decisions

!
No auto-expand on hover. With chat at 320px, accidental expansion causes layout jank. Click-to-expand or hover-flyouts for individual sections (like Slack). Commit to 56px as the default.
!
Auto-expand only in Settings. Settings view has no chat context needed — the viewport has room. The expanded sidebar helps with settings section navigation. Everywhere else: 56px.
!
Three core nav items max. Dashboard, Tasks & Projects, Agents & Automations. Templates live in Gallery. Workflows live inside Projects. Every extra icon is cognitive load at 56px. Resist the urge to add more.
!
Agent status is ambient. Green pulsing dot on the Agents icon when any agent is running. Users need to know "something is working in the background" without clicking in. This is subtle but important for trust in AI delegation.
