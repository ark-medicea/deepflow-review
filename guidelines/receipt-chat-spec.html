Implementation Specification

Receipt Chat

A thermal-receipt-printer-inspired chat panel. Monospace text on dark paper. Not a messaging app — a command receipt unrolling from a machine.

🧾 TL;DR — What This Is
  • An alternate chat panel visual style for DeepFlow — receipt printer aesthetic, monospace typography, dark paper background
  • Sits inside the existing resizable panel system (280–400px, sidebar → chat → content → detail)
  • Behaviour and architecture from the Chat Architecture spec are unchanged — this spec covers visual design and React components only
  • Seven message types: user, machine, thinking disclosure, tool call disclosure, task confirmation, error, status lists
  • Thread system with paper-fade-to-dark transitions between conversation sessions
  • Complete CSS token specification — every colour, size, and timing extracted from the working prototype
  • Full React component API with TypeScript types — ready for implementation
01 — Philosophy

This Is Not a Messaging App

It's a receipt printer. Old accountant thermal paper. Continuous tape, monospace type, minimal chrome.

The chat reads like a printed receipt unrolling from a machine. Text on paper. Nothing else.
🖨️
Text-First

No bubbles, no avatars in messages, no rounded cards. Just text on paper. Hierarchy through colour, borders, and indentation — not size.

⬛
Dark Paper

Dark mode with a slightly lighter "paper" background. The contrast creates the illusion of paper sitting on a dark desk. Receipt tape, not chat window.

📐
Full Width

Fills the entire resizable panel. No centred strip. Content has a max-width for readability (720px) but the container is edge-to-edge.

05 — Threads

Thread Breaks & Headers

Conversations are separated by a five-part visual break — the receipt paper fading out, a dark gap, a dateline, and new paper fading in.

Based on previous imports, about 4–5 minutes.

Mon, 23 Mar · 11:42

Mon, 23 Mar · 11:42
Quick update: Q4 actuals imported successfully.
11:42
Thread Break Structure (top → bottom)
# Element Height Background
1 Fade out 48px linear-gradient(to bottom, var(--rc-paper), var(--rc-bg-dark))
2 Dark gap 24px var(--rc-bg-dark) solid
3 Dateline rule auto Dark bg + 1px #252a35 lines + centred timestamp
4 Small gap 16px var(--rc-bg-dark) solid
5 Fade in 48px linear-gradient(to bottom, var(--rc-bg-dark), var(--rc-paper))

Critical: Thread break bleeds to panel edges with margin: 0 -20px (negating the receipt tape padding).

Thread Header

⬡ hexagon: cyan, opacity: 0.6, font-size: 11px
"DEEPFLOW": text-transform: uppercase, font-size: 10px, letter-spacing: 2.5px, color: var(--rc-text-muted), font-weight: 300
Dateline: decorative gradient dashes (max-width 60px each side, opacity: 0.3)

When Threads Break

User action: explicit "new thread" button
Time gap: configurable threshold (default: 2+ hours since last message)
Session start: new login or session

TypeScript Types (Key Interfaces)
type MessageType = 'user' | 'machine';
type MessageVariant = 'default' | 'error';
type ChipVariant = 'cyan' | 'amber' | 'grey';

interface ChatMessage {
  id: string;
  type: MessageType;
  variant?: MessageVariant;
  content: string;
  timestamp: string;              // ISO 8601
  disclosures?: Disclosure[];
  taskConfirmations?: TaskConfirmation[];
  statusItems?: StatusItem[];
}

interface Disclosure {
  id: string;
  kind: 'thinking' | 'tool';
  label: string;
  content: string;
  defaultOpen?: boolean;          // default: false
}

interface TaskConfirmation {
  id: string;
  title: string;
  url: string;
  status: 'complete' | 'running';
}

interface StatusItem {
  text: string;
  status: 'done' | 'running' | 'default';
}

interface ShortcutChip {
  id: string;
  label: string;
  variant: ChipVariant;
  action: string;
}

interface ChatThread {
  id: string;
  startedAt: string;
  messages: ChatMessage[];
}

08 — Accessibility

ARIA, Contrast & Keyboard

Receipt aesthetics don't excuse poor accessibility. Every element must be keyboard-navigable and screen-reader-friendly.

🔘
Disclosures

<button> semantics.
aria-expanded="false|true"
aria-controls → content ID
Content: display: none when collapsed

⏰
Timestamps

Use <time datetime="..."> elements for all dates and times.
Datelines and message timestamps.

⌨️
Keyboard

Chips: arrow keys + Enter/Space
Disclosures: Tab + Enter/Space
Input: Tab focus, Enter sends
Chat region: role="log", aria-live="polite"

Colour Contrast Ratios
Element Foreground Background Ratio Grade
Primary text #c8cdd5 #1a1f2a ~7.2:1 AAA ✓
Dim text #8b919a #1a1f2a ~4.5:1 AA ✓
Cyan accent #00D4F5 #1a1f2a ~7.8:1 AAA ✓
Green (success) #4ade80 #1a1f2a ~8.5:1 AAA ✓
Red (error) #f87171 #1a1f2a ~6.7:1 AA ✓
Muted text (timestamps) #555b65 #1a1f2a ~2.5:1 Below AA ⚠

Muted text (#555b65) is used for timestamps only — supplementary information that's also communicated by position and format. Consider bumping to #6b7280 (~3.5:1) for better compliance.

09 — Integration

Fitting Into DeepFlow

How the receipt chat connects to the existing panel system, WebSocket events, and chat history API.

Panel System

Replaces <ChatPanel /> in the shell layout with <ReceiptChat />.

Uses react-resizable-panels: min 280px, max 400px, default 320px.

Double-click handle to collapse. Size persisted via Zustand → IndexedDB.

WebSocket Events

chat.message { role, content, metadata }

role: 'user' → <Message type="user">
role: 'assistant' → <Message type="machine">
metadata.thinking → <ThinkingDisclosure>
metadata.toolCalls[] → <ToolCallDisclosure>
metadata.error → variant="error"

Chat History Pagination

Initial load: latest thread (N messages).
Scroll to top → onScrollTop → fetch previous thread from API.
Older threads prepended. Thread breaks inserted by time gap.
TanStack Query: staleTime: 5min, gcTime: 30min.

Context Scoping

Dashboard: "Ask about your workflow…"
Graph View: "Ask about [Project]…"
Workspace: "Ask about this task…"

Chips update per context. Context change may trigger thread break.

10 — Anti-Patterns

Do NOT Do These

Receipt chat has a very specific aesthetic. These would break it.

❌ Don't Why
Chat bubbles Messages are flat text on paper, not in rounded rectangles
Avatars in messages No user/bot icons next to messages — role is shown by border + prompt
Right-aligned user messages Everything is left-aligned. This is a receipt, not iMessage
Background colours on messages Only exception: error variant's subtle red tint
Sans-serif fonts Everything in the chat panel is monospace. No exceptions
Emoji reactions This is a receipt, not Slack
Animated typing dots Use "Thinking…" disclosure that converts to content
Grouped timestamps Each message gets its own timestamp
